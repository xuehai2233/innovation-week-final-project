### CardBundle 重构 ✅ 已完成

**问题**：`_pick_best_card()` 对同名卡只返回一个 chunk，导致 Follower（进化前）和 FollowerEvo（进化后）两者只能保留其一。两者效果完全不同（入场曲/UB vs 进化时/超进化），丢失任意一半都会导致错误裁定。

**方案**：从 `name_cn -> best_chunk` 改为 `name_cn -> CardBundle{follower, follower_evo, ...}`

**改动文件**:

**retriever_local.py**：
- `_card_name_index` → 拆分为 `_card_bundles` + `_card_names`
- 新增 `_build_card_meta()` 工厂函数
- `_build_card_name_index()` 重写：按 (name_cn, card_type) 分组构建 Bundle
- 新增 `_dedup_same_type()`：同 card_type 内去重（多版本稀有度选最佳），不同 card_type 绝对不去重
- `_pick_best_card()` → 移除（职责拆分到 `_dedup_same_type`）
- `_fuzzy_match_card()` → `_fuzzy_match_card_name()`：只返回 name_cn 字符串
- `search_cards_by_name()` 重写：返回 CardBundle 列表而非 flat meta 列表

**rag_pipeline_local.py**：
- `extract_card_names`：`_card_name_index.keys()` → `_card_bundles.keys()`
- 新增 `_detect_card_question_focus(question)`：检测进化后/进化前关键词，返回 evo_first/base_first/balanced
- `build_prompt()` 重写：
  - 参数 `exact_card_matches` → `card_bundles`（CardBundle 列表）
  - 卡牌区按 bundle 展示：`=== 进化前 (Follower) ===` / `=== 进化后 (FollowerEvo) ===` 分区
  - 根据 `_detect_card_question_focus` 决定展示顺序
  - 非随从卡（Spell/Amulet）直接展示
  - 向量检索补充卡牌去重后独立展示
- `judge()` 重写：bundle → flat sources 转换用于 API 响应

**test_follower_evo_bundle.py**（新增）：
- 测试1：CardBundle 结构验证（可可萝=双形态，安&古蕾娅=仅Follower，灼热风暴=Spell，灭剑焰龙=仅Evo）

### 修复 V4-Pro Thinking 模式空回答 ✅ 已完成

**问题**：同一道问题前几次"未返回有效回答"，最后一次有效。根因是 V4-Pro Thinking 模式消耗 token 太多：prompt(~1500 tokens) + max_tokens=2048，Thinking 推理耗光配额后 content 返回空。

**修复**（`backend/local_llm.py`）：
- `max_tokens`: 2048 → 8192
- `generate()` 新增重试逻辑：content 为空时自动 fallback 到不带 thinking 的纯回答模式
- `_generate_simple()`：新增纯回答 fallback 方法
- 异常时同样走 fallback 重试
