### 阶段八：修复卡牌效果截断问题 ✅ 已完成（2026-06-02 17:00）
- **问题**：卡名匹配命中后，`_format_card_for_judge()` 的 `extract_field("中文效果")` 使用 `([^\n]+)` 只匹配单行，导致多行效果被截断。例如「马纳历亚的亲爱挚友·安＆古蕾娅」有3条效果（疾驰、结束阶段召唤、入场曲），只展示了第一条，LLM 因此误判"没有入场曲"。
- **根因**：`_format_card_for_judge` 中的 `extract_field` 函数正则 `([^\n]+)` 只匹配单行
- **修复**：
  1. `_format_card_for_judge` 重写：新增 `_extract_multiline_field()` 支持多行效果提取；效果文本按换行拆分并编号展示；输出格式改为多行结构化
  2. `build_prompt`：卡牌区增加提示"以下卡牌文本为完整效果文本，请阅读全部效果后再裁定"
  3. `retriever_local.py`：`_build_card_name_index` 和 `search_cards_by_name` 增加 `is_exact_card_match` 和 `is_full_card_text` 标记
  4. `extract_card_names`：增加子串扫描 fallback，处理无括号嵌入的卡名（如"马纳历亚的亲爱挚友·安＆古蕾娅"直接出现在自然语言中）
  5. `judge()` 的 sources 格式化：卡牌 source 增加完整性标记，不完整时加 warning
  6. 新增 `test_card_full_text.py`：7项测试全部通过，验证完整效果文本（疾驰、结束阶段、安的巨大英灵、入场曲、舍弃2张、5点伤害、抽取2张卡）
- **改动文件**：`backend/rag_pipeline_local.py`（核心），`backend/retriever_local.py`（标记），`test_card_full_text.py`（新增）
- **期望裁定**：手牌只剩1张时不能发动入场曲，因为入场曲要求舍弃手牌中2张学院类型卡片
- ChromaDB 1.5.x 无法读取旧版 HNSW 索引（版本不兼容）
- 决策：复用 tfidf_encoder.pkl 同样的 TF-IDF+SVD 方案，从 rag_chunks/ 重新构建 FAISS 索引
- 执行：build_vector_db_local.py 成功构建 14,010 条向量（384维），耗时 97 秒
- 修复：脚本添加 UTF-8 编码输出避免 Windows GBK 问题
- FAISS 索引位置：sve_faiss_index/（~119MB）
